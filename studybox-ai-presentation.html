03. 전체 동작 흐름

04 / 08

질문부터 답변 저장까지,
안전하게 연결됩니다.

STEP 1질문 입력모드 · 학년 · 길이 선택
STEP 2자체 API 요청브라우저 → 백엔드
STEP 3인증·한도 확인사용자·요청량 검증
STEP 4AI 답변 생성프롬프트·대화 맥락 전달
STEP 5검증·저장·표시구조화한 결과를 화면에 출력
핵심 원칙: 브라우저는 외부 AI API에 직접 연결하지 않습니다.

04. 사용한 API

05 / 08

OpenRouter의
OpenAI 호환 API를 사용했습니다.

AI 답변 생성 API

OpenAI와 같은 형식의 Chat Completions 요청을 보내 AI 모델의 답변을 생성합니다. 현재 기본 모델은 Qwen 모델이며, 실패 시 대체 모델을 순서대로 시도합니다.

POST /chat/completions
AI_PROVIDER: openai-compatible
AI_BASE_URL: openrouter.ai/api/v1
기본 모델: Qwen 3 Next
대체 모델: Gemma · GPT-OSS · DeepSeek 등
API 키는 브라우저가 아닌 서버 환경의 Secret으로 관리합니다.

05. AI 답변 설계

06 / 08

질문만 보내지 않고,
학습 맥락까지 전달합니다.

서버는 시스템 프롬프트와 이전 대화 내용, 사용자가 선택한 학습 설정을 함께 AI에 전달합니다.

  • 학습 모드별 답변 전략을 지시
  • 학년 수준에 맞는 어휘와 깊이 조절
  • 이전 대화 맥락을 반영한 후속 질문 처리
  • 완성 답안 대필보다 이해와 자기 주도 학습 유도

응답 형식

구조화된 JSON

AI는 항상 같은 형식으로 답변하도록 요청받습니다. 서버는 받은 데이터를 다시 검증한 뒤 화면에 표시합니다.

title · 답변 제목
summary · 핵심 요약
sections · 단계별 설명
일관된 형식 덕분에 답변을 더 읽기 쉽게 보여 줄 수 있습니다.
